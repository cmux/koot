## koot.js 的定位
* 追求极致的用户体验及开发体验的完整的前端解决方案

## koot.js 的能力
* 默认支持同构、SPA多种输出
* 极致的按需加载
* 无需配置，全自动的 webpack 集成开发环境，并支持自定义

## 开发规范

### 命名规范
* 要努力推敲每个名称和其实际做的事情是否一致，也就是命名的准确性；
* 好的命名自带注释属性;

#### 目录
英文全部小写，连字符使用 '-'。（例如：some-component）;

#### 文件
英文全部小写，连字符使用 '-'。（例如：some-component.jsx）;

#### 变量
不同语言有不同的规范;
变量命名推荐两种规范：
* 下划线命名：some_variable
* 驼峰式命名：someVariable

#### Javascript 变量
推荐驼峰式

#### css 样式
因为 css 本身就是用中划线作为连接符
推荐使用中划线作为连接符（例如：font-family）
